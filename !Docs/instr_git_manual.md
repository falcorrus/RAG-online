# Инструкция по работе с ветками Git

## 1. Создание новой ветки
Используется для разработки новых фич или исправлений, чтобы не ломать основной код.
```bash
git checkout -b имя-ветки
```

## 2. Переключение между ветками
Вернуться в основную ветку:
```bash
git checkout master
```
Перейти в созданную ветку:
```bash
git checkout имя-ветки
```

## 3. Слияние изменений (Merge)
Когда работа в ветке завершена и проверена, её нужно объединить с основной:
1. Перейдите в master: `git checkout master`
2. Обновите master (если нужно): `git pull origin master`
3. Влейте ветку: `git merge имя-ветки`

## 4. Удаление ветки
После успешного слияния ветку можно удалить:
```bash
git branch -d имя-ветки
```

## 5. Пуш ветки в облако (без деплоя)
Чтобы сохранить ветку на GitHub как черновик, не обновляя живой сайт:
```bash
git push origin имя-ветки
```

---
**Помни:** В этом проекте `./deploy.sh` работает с веткой `master`. Все изменения, которые должны попасть на сайт, должны быть сначала влиты в `master`.
