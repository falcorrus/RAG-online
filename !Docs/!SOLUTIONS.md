# Решенные проблемы в RAG-online

## Проблема: 413 Request Entity Too Large при загрузке базы знаний
- **Описание:** При попытке загрузить файл `FF-BASE_ALL.md` (2МБ) через админку RAG-online сервер возвращал ошибку 413.
- **Причина:** Лимит Nginx `client_max_body_size` по умолчанию составляет 1МБ. При переходе на HTTPS через Certbot, настройки часто не копируются автоматически в блок 443, если не заданы глобально.
- **Решение:**
    1. Добавлен параметр `client_max_body_size 50M;` в шаблон конфигурации Nginx (`setup_ssl.sh`).
    2. Глобально на VPS в `/etc/nginx/nginx.conf` добавлен лимит в секцию `http`.
    3. Выполнен перезапуск Nginx: `sudo nginx -t && sudo systemctl reload nginx`.

## Проблема: Gemini видит только начало большой базы знаний
- **Описание:** Даже при успешной загрузке, ИИ отвечал только по началу файла.
- **Причина:** В `server.py` был жесткий лимит `MAX_TOKENS = 10000`, что для современных моделей крайне мало.
- **Решение:** Увеличен лимит `MAX_TOKENS` до **300 000** и `limited_content` для генерации подсказок до **20 000**. Это позволяет Gemini 2.0 Flash обрабатывать всю базу целиком.
- **Rationale:** Увеличение окна контекста позволяет использовать возможности Long Context моделей Gemini без потери данных при индексации (поскольку здесь используется простой RAG на основе полного контекста).

## Проблема: Жестко заданные префиксы и авто-извлечение подписи в ответах
- **Описание:** В интерфейсе подпись к ответам всегда содержала префикс "Для вопросов:" и извлекалась только автоматически из базы знаний ИИ.
- **Решение:**
    1. Обновлена модель `TenantSettings` в `server.py` для поддержки поля `underAnswerText`.
    2. В `index.html` добавлено текстовое поле (max 50 символов) в админ-панель.
    3. В `script.js` реализована логика: если пользователь ввел текст вручную, он отображается "как есть", без префиксов.
    4. Обновлена API-ручка `get_public_settings`: она сначала ищет ручную настройку, и только если её нет, берет результат из AI-кэша.
- **Rationale:** Гибридное управление (Override Pattern) дает пользователю контроль над важными элементами брендинга, сохраняя автоматизацию как фолбек (fallback) для новых пользователей.
