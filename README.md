#  RAG Online 🤖

Легкая и красивая веб-база знаний с умным поиском на основе **Google Gemini 2.0 Flash**.
Проект позволяет создать персональный AI-ассистент для вашего бизнеса за 5 минут.

## ✨ Возможности

*   **Умный поиск (RAG):** ИИ анализирует вашу базу знаний и отвечает на вопросы естественным языком.
*   **Автоматическая настройка:** Название проекта и контакты извлекаются автоматически из базы знаний.
*   **Мультиязычность:** Полная поддержка **RU, EN, PT**. Интерфейс и ответы адаптируются под язык пользователя.
*   **Умная подпись:** Контактные данные отображаются в специальной плашке под ответом и автоматически попадают в буфер обмена при копировании.
*   **Glassmorphism UI:** Современный интерфейс с темной темой, анимациями и адаптивностью.
*   **Уведомления:** Мгновенные оповещения о новых регистрациях в Telegram.

## 🚀 Архитектура

*   **Frontend:** Vanilla JS, HTML5, CSS3.
*   **Backend:** Python 3.12, FastAPI, Uvicorn.
*   **Database:** JSON-хранилище для настроек и сессий пользователей.
*   **AI:** Google Gemini API (модель 2.0 Flash).

## 📝 Как подготовить базу знаний (.md)

Для автоматической настройки интерфейса добавьте в начало вашего файла раздел `# Общие настройки`:

```markdown
# Общие настройки

## Название: 
Моя Компания

## Подпись: 
telegram @my_contact
```

*   **Название** станет заголовком сайта.
*   **Подпись** появится внизу ответов в блоке «Для вопросов: ».
*   Данные будут автоматически переведены на RU, EN, PT при загрузке.

## ⚙️ Настройки и Установка

### 1. Локальный запуск

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/falcorrus/RAG-online.git
    cd RAG-online
    ```

2.  Настройте окружение (`.env`):
    ```env
    GEMINI_API_KEY=your_key
    TELEGRAM_BOT_TOKEN=your_bot_token
    TELEGRAM_CHAT_ID=your_id
    ```

3.  Запуск:
    ```bash
    pip install -r requirements.txt
    python3 -m uvicorn server:app --host 0.0.0.0 --port 8006 --reload
    ```

### 2. Деплой на VPS

#### Боевой сервис (Production)
Деплой автоматизирован через скрипт `deploy.sh`:
```bash
./deploy.sh "ваше сообщение"
```
Сайт: [https://rag.reloto.ru](https://rag.reloto.ru)

#### Тестовый сервис (Staging/Dev)
Для проверки изменений без риска для основного сайта используйте:
```bash
./deploy-dev.sh "ваше сообщение"
```
Сайт: [https://dev.rag.reloto.ru](https://dev.rag.reloto.ru)

**Схема работы деплоя:**
1.  **Локальный коммит:** Скрипт фиксирует все изменения.
2.  **Push на VPS:** Код отправляется в соответствующий репозиторий (`vps` для продакшена, `dev` для тестов).
3.  **Обновление файлов:** На сервере срабатывает Git-hook, который обновляет рабочие файлы в нужной папке (`/opt/RAG-online` или `/opt/RAG-online-dev`).
4.  **Перезапуск:** Соответствующий сервис (`rag-online` или `rag-online-dev`) перезапускается автоматически.

**Если VPS не обновился (ручная синхронизация):**
Если пуш на VPS завершился ошибкой или код на сайте остался старым, выполните:
```bash
ssh root@server.reloto.ru "cd /opt/RAG-online && git fetch origin && git reset --hard origin/master"
```

**Откат к предыдущим версиям:**
На VPS хранится история всех деплоев. Чтобы вернуться к старой версии:
1.  Посмотрите список последних "точек":
    ```bash
    ssh root@server.reloto.ru "cd /opt/RAG-online && git log --oneline -n 10"
    ```
2.  Откатитесь к нужной точке (замените `hash` на код коммита):
    ```bash
    ssh root@server.reloto.ru "cd /opt/RAG-online && git reset --hard hash"
    ```

*Примечание: GitHub является основным архивом проекта. Всегда синхронизируйте локальную копию с GitHub перед серьезными изменениями.*

## 🛠 Технические особенности

*   **Язык клиента:** Определяется автоматически при первом запуске (browser language), ручной выбор сохраняется в `localStorage`.
*   **Фильтрация контекста:** Раздел «Общие настройки» вырезается из данных, отправляемых ИИ для ответа, чтобы избежать дублирования контактов в тексте.
*   **Безопасность:** JWT-токены для сессий, bcrypt для паролей, изоляция баз знаний между пользователями.
*   Внедрена отказоустойчивая обработка (fallback) на случай отсутствия файла на диске: если файл не будет прочитан, сервер
          автоматически использует надежный дефолтный промпт прямо из кода.
